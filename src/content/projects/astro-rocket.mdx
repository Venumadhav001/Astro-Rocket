## What I Built

Astro Rocket is a complete, launch-ready website theme built on Astro 6 and Tailwind CSS v4. It ships as a fully styled, fully wired-up site — a homepage, blog, projects section, about page, and contact form — all ready to go live the moment you swap in your own text. It's built for web designers, developers, bloggers, and anyone who needs a polished online presence without starting from scratch. Clone it, configure ten lines in `site.config.ts`, and your site is done.

## Live Theme Switching. No Rebuild Required.

Most Astro themes give you one colour and a note in the README explaining how to dig into a CSS file and rebuild to change it. Astro Rocket ships 12 ready-made colour themes — Orange, Amber, Lime, Emerald, Teal, Cyan, Sky, Blue, Indigo, Violet, Purple, and Magenta — all selectable with a single click from the header.

The moment you click a swatch, every brand-coloured element on the page updates live: the logo badge, the blog image gradients, every button, every link, every accent. No file edits, no server restart, no waiting. The entire colour system is built on CSS custom properties, so the browser handles it instantly via the cascade. Once you've settled on a colour, you delete one prop from the layout file and the selector disappears — clean and gone.

## Auto-Generated Branding

There's no logo file to replace. Astro Rocket generates the logo badge at runtime from the first letter of your site name and the active brand colour. Change the name in `site.config.ts` and the badge updates everywhere — header, footer, structured data. The favicon works the same way: it's an SVG rendered at build time from the same letter and colour. New site name, instant new favicon. No Figma, no Illustrator, no fiddling with `public/`.

## SEO — The Complete Stack

Astro Rocket doesn't ship "three meta tags and call it SEO." It ships a complete implementation covering every layer.

- **Meta tags** — title, description, and canonical URL on every page, automatically
- **Open Graph + Twitter Cards** — complete social metadata; blog posts use their cover image, other pages use the default OG image
- **JSON-LD structured data** — WebSite, Person, Organization, ProfessionalService, BlogPosting, Breadcrumb, and FAQ schemas, all populated from `site.config.ts` and post frontmatter
- **Auto-generated sitemap** — `/sitemap-index.xml` includes every page Astro renders, always in sync
- **Dynamic robots.txt** — with sitemap reference, generated at build time
- **Per-page noindex/nofollow** — pass a prop to any layout to exclude a page from crawling
- **Search console verification** — Google and Bing codes go in `.env`, rendered in `<head>` automatically

Every structured data value comes from your config. There's nothing hardcoded to find and replace.

## The Interactions

**Scroll progress bar.** A 2px brand-coloured line at the header edge fills from left to right as you scroll. It uses `requestAnimationFrame` throttling so the fill tracks your position directly with no lag. It's enabled on three page types — homepage (above the floating header), blog index, and individual posts — and the position is independently configurable per page. Switch themes and the bar colour changes instantly.

**Scroll progress ring.** The back-to-top button has a circular SVG arc that fills clockwise as you read. It's built from two circles — a faint track and a brand-coloured arc animated via `stroke-dashoffset`. No JavaScript needed for the colour: it uses `stroke="currentColor"` with `class="text-brand-500"`, so the theme cascade handles it automatically.

**Animated typing effect.** The hero headline cycles through words with a natural typing and deleting rhythm. Three non-obvious problems were solved to make it production-quality: client-side navigation re-firing (handled by `astro:page-load`), layout shift from words of different lengths (solved by measuring every word before the animation starts and locking `minWidth` to the widest), and descender clipping (fixed by removing `overflow: hidden`).

**Scroll-reactive header.** The floating homepage header is transparent at the top of the page, sitting cleanly over the hero gradient. The moment you scroll past 60px, it gains a solid background and the text flips to normal colours — all driven by a `data-scrolled` attribute toggle and CSS transitions. No layout shift, no JavaScript style manipulation.

**Page transitions.** Every internal link uses Astro's built-in View Transitions API for a smooth page-to-page fade. One line of config in `BaseLayout.astro`, zero per-page setup. The typing effect, back-to-top ring, and all other scripts correctly clean up and reinitialise on each navigation via `astro:page-load` and `astro:before-swap`.

## Dark Mode — Done Right

The site is dark by default. Not as a fallback — as a deliberate design choice. The typography, contrast ratios, and hero gradient were all designed dark-first.

The implementation uses `sessionStorage` rather than `localStorage`. The distinction matters: `localStorage` remembers the user's last choice forever, which means a returning visitor might see the wrong mode if they poked the toggle once. `sessionStorage` resets on every new tab, so every first impression is the dark design as intended. The preference is still respected for the duration of the visit — it just doesn't persist past the tab.

The dark class is baked into the server-rendered HTML, and an inline script runs before first paint to remove it only if the visitor chose light in this session. There is no flash in either direction.

## Zero JavaScript by Default

Astro Rocket ships zero JavaScript to the browser by default. Every page is server-rendered HTML with CSS-driven animations. The islands architecture means JavaScript is loaded only for the specific components that need it — the typing effect, the theme selector, the contact form — and only on the pages where those components appear.

The result: Lighthouse 95+ across Performance, Accessibility, Best Practices, and SEO. Not a one-time measurement — a structural property of how the site is built.
